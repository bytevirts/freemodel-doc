---
title: "Gemini generateContent"
description: "Use the Gemini generateContent-compatible endpoint."
api: "POST /v1beta/models/{model}:generateContent"
---

<Warning>
  This is a compatibility endpoint for existing Gemini-style integrations. For new integrations, use the [OpenAI-compatible API](/api-reference/chat-completions).
</Warning>

Authenticate with a Bearer key. The gateway also accepts a FreeModel key in the `key` query parameter for Gemini-style clients.

<ParamField path="model" type="string" required>
  A Gemini-compatible model ID returned by
  [`GET /v1/models`](/api-reference/list-models).
</ParamField>

<ParamField body="contents" type="array" required>
  Gemini-style content objects containing `parts`.
</ParamField>

<ParamField body="generationConfig" type="object">
  `generationConfig.temperature` (number) controls sampling variation;
  `generationConfig.maxOutputTokens` (integer) caps generated tokens when
  supported.
</ParamField>

<Note>
  Model availability can change. The [models page](https://freemodel.app/models) lists the current catalog and pricing.
</Note>

<RequestExample>
```bash Curl
curl "https://api.freemodel.app/v1beta/models/<gemini-model-id>:generateContent" \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [
      {
        "role": "user",
        "parts": [
          {"text": "Explain an API gateway in one sentence."}
        ]
      }
    ],
    "generationConfig": {
      "temperature": 0.2,
      "maxOutputTokens": 100
    }
  }'
```
</RequestExample>
