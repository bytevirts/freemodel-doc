---
title: "Claude messages"
description: "Use the Anthropic Messages-compatible endpoint."
api: "POST /v1/messages"
---

<Warning>
  This is a compatibility endpoint for existing Anthropic-style integrations. For new integrations, use the [OpenAI-compatible API](/api-reference/chat-completions).
</Warning>

Choose a Claude-compatible model ID returned by [`GET /v1/models`](/api-reference/list-models). FreeModel accepts a standard Bearer key; integrations that send an `x-api-key` header can also use a FreeModel `sk-...` key.

<ParamField body="model" type="string" required>
  A Claude-compatible model ID available to your key.
</ParamField>

<ParamField body="messages" type="array" required>
  Anthropic-style conversation messages.
</ParamField>

<ParamField body="max_tokens" type="integer" required>
  Maximum tokens to generate.
</ParamField>

<ParamField body="stream" type="boolean">
  Set to `true` when your client expects an event stream.
</ParamField>

<ParamField body="temperature" type="number">
  Sampling variation when supported by the selected model.
</ParamField>

<Tip>
  With an Anthropic SDK, set its base URL to `https://api.freemodel.app`. See [Anthropic SDK](/sdks/anthropic-sdk).
</Tip>

<RequestExample>
```bash Curl
curl https://api.freemodel.app/v1/messages \
  -H "Authorization: Bearer sk-xxxx" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "<claude-model-id>",
    "max_tokens": 128,
    "messages": [
      {
        "role": "user",
        "content": "Explain what an API gateway does in one sentence."
      }
    ]
  }'
```
</RequestExample>

<ResponseExample>

```json Response
{

    }

    "input_tokens": 12,
    "output_tokens": 15
  }
}
```
</ResponseExample>
