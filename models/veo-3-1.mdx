## Generate from text

```bash
curl https://api.freemodel.app/v1/videos \
  -H "Authorization: Bearer $FREEMODEL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "veo3-1-lite-4k",
    "prompt": "A coastal train crosses a stone bridge at sunrise, synchronized waves and rail sound",
    "seconds": "8",
    "aspect_ratio": "16:9"
  }'
```

## Guide the start and end frames

Add one or two public `image_urls` to the Lite, Fast, or Quality routes. The first URL is the start frame. When you provide two URLs, the second is the end frame.

```bash
curl https://api.freemodel.app/v1/videos \
  -H "Authorization: Bearer $FREEMODEL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "veo3-1-fast",
    "prompt": "Move smoothly from the supplied opening frame to the closing frame",
    "seconds": "8",
    "aspect_ratio": "16:9",
    "image_urls": [
      "https://example.com/start.jpg",
      "https://example.com/end.jpg"
    ]
  }'
```

`veo3-1-fast-ref` is different: it requires one to three public image URLs. Use it when visual references are the primary source of identity or style guidance.

## Poll and download

Save the returned `id`. Include the same model ID while polling and downloading:

```bash
curl "https://api.freemodel.app/v1/videos/$VIDEO_ID?model=veo3-1-lite-4k" \
  -H "Authorization: Bearer $FREEMODEL_API_KEY"

curl -L "https://api.freemodel.app/v1/videos/$VIDEO_ID/content?model=veo3-1-lite-4k" \
  -H "Authorization: Bearer $FREEMODEL_API_KEY" \
  --output video.mp4
```

See [Video generation](/api-reference/video-generation) for the full asynchronous workflow and response shapes.
